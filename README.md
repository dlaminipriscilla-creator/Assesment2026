#  Advanced Phone Order Form Automation Project

## Project Overview

This project is created to automate the process of completing an advanced phone order form on a website.

The goal of this project is to verify that users can successfully enter customer details, select phone options, complete form fields, and submit an order flow while ensuring the application behaves as expected.

This project demonstrates my practical experience with **UI automation testing, element identification, XPath creation, and test automation using IntelliJ IDEA.**

---

## 🛠️ Tools & Technologies Used

* **Java** – Programming language
* **Selenium WebDriver** – Web automation framework
* **IntelliJ IDEA** – Development environment
* **TestNG** – Test execution framework
* **Maven** – Project dependency management
* **Git & GitHub** – Version control and project sharing

---

## ✅ Automated Test Scenario

The automation covers:

✔ Opening the website
✔ Navigating to the phone ordering section
✔ Filling in an advanced order form
✔ Selecting phone-related options
✔ Entering customer information
✔ Interacting with web elements
✔ Validating the ordering workflow

---

## 🔎 Locators & Automation Techniques

During this project, different locator strategies were explored, including:

* ID locators
* XPath locators
* Dynamic XPath handling
* Relative XPath
* Using attributes and text values to create stable locators

Special attention was given to creating reliable locators that avoid dependency on changing dynamic values.

---

## Testing Approach

The automation follows a structured approach:

1. Identify test steps from the user journey
2. Inspect web elements using browser developer tools
3. Create stable locators
4. Automate user interactions using Selenium WebDriver
5. Execute tests and verify expected behaviour
